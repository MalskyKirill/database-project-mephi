# Курсовой проект по Базам Данных НИЯУ МИФИ

## Краткое описание проекта
`database-project-mephi` — это учебный Java-проект на JDBC и MySQL, который демонстрирует работу с реляционными БД через набор прикладных кейсов. Приложение поднимает схемы данных из SQL-скриптов, наполняет их тестовыми данными и выполняет аналитические SQL-запросы с выводом результатов в консоль.

## Цель проекта
Показать практическое применение SQL (JOIN, CTE, рекурсия, агрегирование, сортировка, фильтрация).

## Структура базы данных
Проект использует 4 независимых схемы (каждая создаётся своим SQL-скриптом):

1. **Транспортные средства (`schema1.sql`)**
    - `Vehicle` — справочник моделей и типа транспорта.
    - `Car`, `Motorcycle`, `Bicycle` — специализированные таблицы с характеристиками.
    - Связь: дочерние таблицы ссылаются на `Vehicle(model)`.

2. **Автомобильные гонки (`schema2.sql`)**
    - `Classes` — классы автомобилей.
    - `Cars` — машины с привязкой к классу.
    - `Races` — гонки.
    - `Results` — результаты машин в гонках.
    - Связи: `Cars.class -> Classes.class`, `Results.car -> Cars.name`, `Results.race -> Races.name`.

3. **Бронирование отелей (`schema3.sql`)**
    - `Hotel` — отели.
    - `Room` — номера отелей.
    - `Customer` — клиенты.
    - `Booking` — факты бронирований.
    - Связи: `Room.ID_hotel -> Hotel.ID_hotel`, `Booking.ID_room -> Room.ID_room`, `Booking.ID_customer -> Customer.ID_customer`.

4. **Структура организации (`schema4.sql`)**
    - `Departments`, `Roles` — справочники отделов и ролей.
    - `Employees` — сотрудники с иерархией через `ManagerID` (self-reference).
    - `Projects` — проекты отделов.
    - `Tasks` — задачи сотрудников в проектах.
    - Связи: внешние ключи между сотрудниками, отделами, ролями, проектами и задачами.

## Реализация проекта
- Главное меню (`DBApp`) для выбора одного из 4 кейсов.
- Загрузка параметров подключения из `src/main/resources/app.properties`.
- Универсальный запуск SQL-скриптов (`SqlScriptRunner`) с транзакцией и rollback при ошибке.
- Выполнение SQL-задач в каждом модуле:
    - `TransportMeansApp`: выборки по фильтрам и `UNION ALL` для разных типов транспорта.
    - `CarRacingApp`: CTE, оконные функции, ранжирование и агрегаты по гонкам/классам.
    - `HotelBooking`: агрегирование бронирований, сегментация клиентов и категоризация отелей.
    - `OrganizationStructure`: рекурсивные CTE по иерархии сотрудников и аналитика по задачам/подчинённым.

## Разбор SQL запросов в кейсах (SQL запросы сформированы в каждом классе)

### 1) Кейс «Транспортные средства» (`TransportMeansApp`)

#### Задача 1
Найдите производителей (maker) и модели всех мотоциклов, которые имеют мощность более 150 лошадиных сил,
стоят менее 20 тысяч долларов и являются спортивными (тип Sport). Также отсортируйте результаты по мощности в порядке убывания.

**SQL решение:**
- Берёт `Vehicle` и `Motorcycle`.
- Соединяет по модели (`JOIN ... ON v.model = m.model`).
- Фильтрует только спортивные мотоциклы: `m.type = 'Sport'`.
- Добавляет ограничения по характеристикам: `horsepower > 150`, `price < 20000`.
- Сортирует по мощности по убыванию.

#### Задача 2
Найти информацию о производителях и моделях различных типов транспортных средств (автомобили, мотоциклы и велосипеды), которые соответствуют заданным критериям.

**SQL решешие:**
- Формирует единый набор из трёх выборок через `UNION ALL`:
    1. Автомобили (`Car`) с фильтром по мощности/объёму/цене.
    2. Мотоциклы (`Motorcycle`) с собственными порогами.
    3. Велосипеды (`Bicycle`) с фильтром по числу передач/цене.
- Для выравнивания структуры результата у велосипедов `horsepower` и `engine_capacity` ставятся `NULL`.
- Добавляется поле `vehicle_type` для маркировки источника строки.
- Итог сортируется по `horsepower DESC`.

### 2) Кейс «Автомобильные гонки» (`CarRacingApp`)

#### Задача 1
Определить, какие автомобили из каждого класса имеют наименьшую среднюю позицию в гонках, и вывести информацию о каждом
таком автомобиле для данного класса, включая его класс, среднюю позицию и количество гонок, в которых он участвовал. Также отсортировать результаты по средней позиции.

**SQL решешие:**
- В CTE `car_stats` считает по каждой машине:
    - среднюю позицию (`AVG(position)`),
    - количество гонок (`COUNT(race)`).
- В CTE `ranked` назначает ранг через `DENSE_RANK()` внутри каждого класса.
- Возвращает машины с `rk = 1` (лучшие в своём классе).

#### Задача 2
Определить автомобиль, который имеет наименьшую среднюю позицию в гонках среди всех автомобилей, и вывести информацию об этом автомобиле,
включая его класс, среднюю позицию, количество гонок, в которых он участвовал, и страну производства класса автомобиля. 
Если несколько автомобилей имеют одинаковую наименьшую среднюю позицию, выбрать один из них по алфавиту (по имени автомобиля).

**SQL решешие:**
- Повторно строит `car_stats`.
- Присоединяет `Classes` для получения страны (`country`).
- Сортирует по средней позиции и берёт одну первую строку `LIMIT 1`.

#### Задача 3
Определить классы автомобилей, которые имеют наименьшую среднюю позицию в гонках, и вывести информацию о каждом автомобиле из этих классов,
включая его имя, среднюю позицию, количество гонок, в которых он участвовал, страну производства класса автомобиля,
а также общее количество гонок, в которых участвовали автомобили этих классов. Если несколько классов имеют одинаковую среднюю позицию, выбрать все из них.

**SQL решешие:**
- Формирует `car_stats` и `class_stats` (средняя позиция по классу).
- В `best_classes` выбирает класс(ы) с минимальной средней позицией.
- Возвращает машины только этих классов + число заездов класса.

#### Задача 4
Определить, какие автомобили имеют среднюю позицию лучше (меньше) средней позиции всех автомобилей в своем классе 
(то есть автомобилей в классе должно быть минимум два, чтобы выбрать один из них). Вывести информацию об этих автомобилях,
включая их имя, класс, среднюю позицию, количество гонок, в которых они участвовали, и страну производства класса автомобиля.
Также отсортировать результаты по классу и затем по средней позиции в порядке возрастания.

**SQL решешие:**
- Для каждого класса считает «среднюю из средних» по машинам и оставляет классы с минимум 2 машинами (`HAVING COUNT(*) >= 2`).
- Возвращает машины, чья средняя позиция лучше среднего по своему классу.

#### Задача 5
Определить, какие классы автомобилей имеют наибольшее количество автомобилей с низкой средней позицией (больше 3.0) 
и вывести информацию о каждом автомобиле из этих классов, включая его имя, класс, среднюю позицию, количество гонок,
в которых он участвовал, страну производства класса автомобиля, а также общее количество гонок для каждого класса.
Отсортировать результаты по количеству автомобилей с низкой средней позицией.

**SQL решешие:**
- В `class_stats` считает:
    - общее число заездов по классу,
    - количество машин с «слабой» средней позицией (`average_position >= 3.0`).
- Оставляет классы, где такие машины есть.
- Возвращает машины с `average_position > 3.0` и статистику класса.

### 3) Кейс «Бронирование отелей» (`HotelBooking`)

#### Задача 1
Определить, какие клиенты сделали более двух бронирований в разных отелях, и вывести информацию о каждом таком клиенте,
включая его имя, электронную почту, телефон, общее количество бронирований, а также список отелей, в которых они бронировали
номера (объединенные в одно поле через запятую). Также подсчитать среднюю длительность их пребывания (в днях) по всем бронированиям. 
Отсортировать результаты по количеству бронирований в порядке убывания.

**SQL решешие:**
- Соединяет `Customer` + `Booking` + `Room` + `Hotel`.
- Считает для клиента:
    - число бронирований,
    - список отелей (`GROUP_CONCAT`),
    - среднюю длительность проживания (`AVG(DATEDIFF(...))`).
- Через `HAVING` оставляет только активных клиентов:
    - более 2 бронирований,
    - более 1 уникального отеля.

#### Задача 2
Необходимо провести анализ клиентов, которые сделали более двух бронирований в разных отелях и потратили более 500 долларов на свои бронирования.

**SQL решешие:**
- Считает суммарные траты клиента (`SUM(r.price)`) и число уникальных отелей.
- Оставляет клиентов с высокой активностью и бюджетом (`SUM(r.price) > 500`).
- Сортирует по возрастанию суммарных трат.

#### Задача 3
Вам необходимо провести анализ данных о бронированиях в отелях и определить предпочтения клиентов по типу отелей.

**SQL решешие:**
- В CTE `hotel_categories` присваивает каждому отелю ценовую категорию по средней цене номера.
- Затем для каждого клиента определяет предпочитаемый тип отеля:
    - приоритет: `Дорогой` > `Средний` > `Дешевый`.
- Дополнительно выводит список посещённых отелей.

### 4) Кейс «Структура организации» (`OrganizationStructure`)

#### Задача 1
Найти всех сотрудников, подчиняющихся Ивану Иванову (с EmployeeID = 1), включая их подчиненных и подчиненных подчиненных, а также самого Ивана Иванова.

**SQL решешие:**
- С помощью рекурсивного CTE `subordinates` строит дерево подчинённых от `EmployeeID = 1`.
- Для каждого найденного сотрудника подтягивает:
    - отдел (`Departments`),
    - роль (`Roles`),
    - список проектов отдела,
    - список персональных задач.

#### Задача 2
Найти всех сотрудников, подчиняющихся Ивану Иванову (с EmployeeID = 1), включая их подчиненных и подчиненных подчиненных,
а также самого Ивана Иванова.

**SQL решешие:**
- Повторяет рекурсивный обход подчинённых.
- Добавляет агрегаты:
    - `TotalTasks` — число задач сотрудника,
    - `TotalSubordinates` — число прямых подчинённых.
- Выводит расширенный управленческий отчёт.

#### Задача 3
Найти всех сотрудников, которые занимают роль менеджера и имеют подчиненных (то есть число подчиненных больше 0).

**SQL решешие:**
- В CTE `manager_tree` строит иерархию подчинённости только для сотрудников с ролью «Менеджер».
- Считает количество подчинённых по всей глубине (`COUNT(DISTINCT SubordinateID)`).
- Оставляет только менеджеров, у которых есть хотя бы один подчинённый (`HAVING > 0`).

## Инструкции по запуску проекта
### 1) Предварительные требования
- JDK 17+
- Maven 3.9+
- MySQL 8+

### 2) Настройка БД
1. Убедитесь, что сервер MySQL запущен.
2. Создайте базы данных:
    - `transport`
    - `racing`
    - `booking`
    - `struct`
3. Отредактируйте настройки в `src/main/resources/app.properties`:
    - `db.user`
    - `db.password`
    - при необходимости URL для каждой БД

Пример создания БД:
```sql
CREATE DATABASE transport;
CREATE DATABASE racing;
CREATE DATABASE booking;
CREATE DATABASE struct;
```

### 3) Сборка
```bash
mvn clean package
```

### 4) Запуск
```bash
mvn exec:java -Dexec.mainClass=DBApp
```

После запуска выберите нужный кейс в меню (1–4). При выборе кейса приложение:
1. Подключится к соответствующей БД.
2. Выполнит SQL-скрипт (создание таблиц и вставка данных).
3. Запустит набор SQL-запросов и выведет результат в консоль.

### Возможные проблемы
- **Ошибка подключения к MySQL**: проверьте `db.user`, `db.password`, URL и доступность сервера.
- **Access denied / Unknown database**: создайте нужные базы и проверьте права пользователя.
- **Порт/хост MySQL отличается**: обновите JDBC URL в `app.properties`.
